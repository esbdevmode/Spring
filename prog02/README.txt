-> POJO vs Java Beans vs Spring Beans
        a. POJO [Plain Old Java Object]
            1. Any object in java is called POJO
            2. No constraints
        b. Java Beans
            1. No - arg Public constructor
            2. To access the property, need to have getter and settter methods
            3. it should implement java.io.serializable
        c. Spring Beans
            1. Any object managed by Spring i,e Spring Beans
            2. Bean Factory or Application Context to manage these spring beans
